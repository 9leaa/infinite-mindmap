# 无限画布思维导图 V16.1

## 1. iPad Chrome 悬浮工具栏修复

截图中工具栏已经显示，但左侧仍显示“正在打开 IndexedDB”。这说明工具栏在应用状态初始化完成前提前出现。此时按钮依赖的 `library/state` 尚未准备好，表现为按钮和拖动同时失效。

V16.1 调整为：

- 当前笔记完成基础初始化后，悬浮工具栏才显示。
- 旧版数据迁移改到后台，并且必须等待连续 5 秒没有用户操作才启动。
- 工具按钮从 `pointerdown` 立即执行，改为稳定的 `pointerup + click`。
- 不再在 `pointerdown` 阶段 `preventDefault()`，避免 iPad WebKit 取消后续点击。
- 拖动只由 `⠿` 专用把手触发。
- 拖动在 `window` 层跟踪 `pointermove/pointerup`，手指移出工具栏也不会丢失。
- 思维导图工具栏和绘图工具栏使用相同的拖动方案。
- 提升工具栏层级并强制启用命中测试。

## 2. 手写周边预渲染

之前手写 Canvas 只等于当前可视工作区大小。移动画布时，屏幕边缘外没有已栅格化内容，因此会出现“移动后旧笔迹才出现”的感觉。

V16.1 在平板上默认预渲染：

- 当前视口；
- 左右各额外 35%；
- 上下各额外 35%。

实际缓存区域约为：

`1.7 × 1.7 = 2.89 个视口面积`

手指移动画布时，周边笔迹已经在 Canvas 中。

为避免 iPad 内存压力：

- 扩展手写 Canvas 在粗指针设备上使用 DPR 1；
- 激光笔 Canvas 不扩大；
- 屏幕外较远笔迹仍通过边界框裁剪；
- 拖动结束后只进行一次最终重绘。

## 3. 启动和迁移

- 当前笔记优先读取并显示。
- 当前笔记显示后，工具栏才开放。
- 旧笔迹、PDF 批注和附件迁移在后台执行。
- 后台迁移检测到 Pencil、手指、鼠标或滚轮操作时会继续等待。
- 每个文档和附件迁移之间会主动让出主线程。
- 若 IndexedDB 长时间未打开，会提示关闭旧版本标签页后刷新。
