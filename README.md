# 无限画布思维导图 V15.8

## 排查结论

“写完后手指很久拖不动画布”主要不是手势判断，而是旧版在落笔后约 320ms 执行重型保存：

1. 深拷贝当前状态。
2. 深拷贝整个笔记库。
3. 重复复制 PDF 的 Base64 字符串。
4. 将整份 PDF 随笔记重新写入 IndexedDB。
5. 同时生成包含全部手写点的撤销快照。

大型 PDF 会让主线程暂时阻塞，因此看起来像手指失效。

## V15.8 存储结构

- `library`：节点、笔迹、批注、附件元数据。
- `binary-assets`：PDF、DOC、PPT 原始 Blob。

日常手写保存不会再读取、复制或重写 PDF 原文件。

旧版本 Base64 附件会在首次打开 V15.8 时自动迁移到 Blob Store。

## 手写和画布移动

- 手写保存改为空闲期执行。
- 完整撤销快照延后到更空闲的时间。
- Pencil 抬起后的触摸保护时间从约 260ms 降至约 48ms。
- 真正手指移动超过约 4px 即开始平移。
- 平移过程中只移动现有手写 Canvas 位图。
- 手指抬起后才完整重绘手写内容一次。

## PDF 性能

- PDF.js 和 Worker 随项目本地部署，不再等待 CDN。
- PDF 直接从 IndexedDB Blob 读取，不再 Base64 解码。
- 页面接近屏幕时才创建 Canvas。
- iPad 同时只渲染 1 页。
- 桌面最多同时渲染 2 页。
- 远离阅读位置的页面 Canvas 自动释放。
- iPad PDF 渲染像素比降为 1，降低像素处理与内存压力。

## 备份

“备份全部笔记”仍包含 PDF 和 Office 原文件。

只有用户主动备份时，应用才读取 Blob 并编码进 JSON，因此大型电子书备份仍需要等待。
